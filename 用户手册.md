## 1. 产品简介

**IE Mode** 是一个 Chrome / Edge 浏览器扩展，点击工具栏按钮后，使用 **Internet Explorer 11（Trident）渲染引擎**在独立窗口中打开当前页面。

适用场景：
- 仅支持 IE 的企业内网系统
- 依赖 ActiveX 控件的遗留 Web 应用
- 需要 IE 兼容模式渲染的旧版网页

> **注意：** 本插件仅支持 Windows 系统。在 macOS / Linux 上点击图标会提示"Windows required"。

## 2. 安装

### 2.1 前置要求

| 要求 | 说明 |
|------|------|
| 操作系统 | Windows 10 / 11 |
| 浏览器 | Chrome 88+ 或 Edge 88+ |
| 权限 | 普通用户权限（无需管理员） |
| .NET | .NET Framework 4.x（Windows 内置，通常已具备） |

### 2.2 加载浏览器扩展

1. 下载并解压项目文件包，确保目录结构完整：

   ```
   IE插件/
   ├── extension/          ← 扩展目录
   ├── native/             ← 原生组件（含 IEModeHost.exe）
   └── install.ps1         ← 安装脚本
   ```

2. 打开浏览器，在地址栏输入：
   - Chrome：`chrome://extensions`
   - Edge：`edge://extensions`

3. 开启右上角的 **开发者模式**（Developer mode）开关。

4. 点击 **加载已解压的扩展程序**（Load unpacked），选择项目中的 `extension/` 文件夹。

5. 扩展加载成功后，工具栏出现 IE Mode 图标。

### 2.3 安装原生组件

原生组件（`IEModeHost.exe`）负责在 Windows 上启动 IE 渲染窗口，需要通过安装脚本注册到系统。

**步骤：**

1. **获取扩展 ID**

   点击工具栏中的 IE Mode 图标，弹窗会显示"Native host not installed"并自动显示当前扩展 ID：

   ```
   ┌─────────────────────────────────────┐
   │ ! Native host not installed         │
   │ ─────────────────────────────────── │
   │ Your extension ID:                  │
   │ [abcdefghijklmnopabcdefghijklmnop] [Copy] │
   │                                     │
   │ 1. Copy the ID above                │
   │ 2. Right-click install.ps1 →        │
   │    Run with PowerShell, paste       │
   │ 3. Reload extension                 │
   └─────────────────────────────────────┘
   ```

   点击 **Copy** 按钮，扩展 ID 即复制到剪贴板（按钮变为绿色"Copied!"表示成功）。

2. **运行安装脚本**

   在项目根目录找到 `install.ps1`，**右键** → **用 PowerShell 运行**。

   脚本提示输入 Extension ID 时，按 **Ctrl+V** 粘贴后回车：

   ```
   Extension ID: abcdefghijklmnopabcdefghijklmnop
   ```

   脚本将自动完成以下操作（无需管理员权限）：
   - 定位 `IEModeHost.exe`
   - 生成 `native/com.iemode.host.json` 配置文件
   - 在注册表 `HKCU` 中注册 Chrome 和 Edge 的 Native Messaging Host

   看到 **Installation complete!** 即表示安装成功，按 Enter 关闭窗口。

   > **高级用法：** 也可在 PowerShell 中直接传入 ID 参数，实现完全无交互安装：
   > ```powershell
   > .\install.ps1 -ExtensionId abcdefghijklmnopabcdefghijklmnop
   > ```

### 2.4 重载扩展

安装脚本执行完毕后，需重载扩展使配置生效：

1. 打开 `chrome://extensions`（或 `edge://extensions`）
2. 找到 IE Mode，点击 **重新加载**（圆形箭头图标）

至此安装完成。

## 3. 日常使用

1. 在浏览器中打开任意 `http://` 或 `https://` 页面。
2. 点击工具栏中的 **IE Mode** 图标。
3. 弹窗短暂显示"Opening in IE mode…"（加载中）。
4. 成功后弹窗显示"Opened in IE mode"并自动关闭，同时弹出独立的 IE 渲染窗口。

> **提示：** 以下类型的页面无法在 IE 模式中打开：`chrome://`、`edge://`、`about:`、`data:`、`javascript:` 等浏览器内部页面。

---

## 4. IE 查看器窗口

成功打开后，会弹出一个独立的 **IE Mode** 窗口，包含简单导航栏：

| 控件 | 功能 |
|------|------|
| ◀ | 后退 |
| ▶ | 前进 |
| 地址栏 | 显示当前 URL，可手动输入新地址 |
| Go | 导航到地址栏中的 URL（也可按 Enter） |

- 窗口标题格式为：`IE Mode — 域名`
- 渲染引擎为 **IE 11**（Trident，注册表 `FEATURE_BROWSER_EMULATION = 11001`）
- 关闭窗口即退出，不影响主浏览器

## 5. 弹窗状态说明

| 状态 | 图标 | 含义 |
|------|------|------|
| Opening in IE mode… | 蓝色转圈 | 正在启动，请稍候 |
| Opened in IE mode | 绿色对勾 | 成功，窗口已打开，弹窗自动关闭 |
| Native host not installed | 红色 ! | 原生组件未安装，按弹窗内步骤安装 |
| Windows required | 红色 ! | 当前系统非 Windows，IE 渲染不可用 |
| IE mode unavailable here | 红色 ! | 当前页面类型（如 chrome://）不支持 |
| Cannot read this page | 红色 ! | 页面 URL 无法读取，尝试刷新后重试 |
| Request timed out | 红色 ! | 原生组件 10 秒内未响应，重试或重装 |
| Native host error | 红色 ! | 原生组件返回错误，见错误详情 |

## 6. 常见问题

**Q：点击图标后一直转圈，超时报错？**
A：原生组件可能崩溃或启动异常。尝试：①重载扩展；②重新运行 install.ps1 确认注册路径正确。

**Q：运行 install.ps1 报"执行策略"错误？**
A：以管理员身份打开 PowerShell，执行：
```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```
然后再运行安装脚本。

**Q：脚本报"IEModeHost.exe not found"？**
A：确认项目文件完整，`native\IEModeHost\bin\Release\IEModeHost.exe` 必须存在。若为源码版本，需先在 Visual Studio 中编译项目（Release 配置）。

**Q：安装完成后重载扩展，点击仍提示"Native host not installed"？**
A：检查以下两点：
① Extension ID 是否填写正确（32位小写字母 a–p）；
② 注册表路径 `HKCU\Software\Google\Chrome\NativeMessagingHosts\com.iemode.host` 是否存在，值是否指向正确的 json 文件路径。

**Q：IE 窗口打开后网页渲染为 IE7 样式？**
A：注册表 `FEATURE_BROWSER_EMULATION` 写入失败（通常是权限问题）。确认当前用户对 `HKCU\SOFTWARE\Microsoft\Internet Explorer\Main\FeatureControl\` 有写权限。

**Q：Mac / Linux 上能用吗？**
A：不能。IE 的 Trident 渲染引擎仅存在于 Windows 系统，在其他系统上插件会提示"Windows required"。

## 7. 卸载

**卸载扩展：**
1. 打开 `chrome://extensions`
2. 找到 IE Mode，点击**移除**

**清理原生组件（可选）：**

以普通用户身份在 PowerShell 中执行：

```powershell
# 删除注册表项
Remove-Item "HKCU:\Software\Google\Chrome\NativeMessagingHosts\com.iemode.host" -Force -ErrorAction SilentlyContinue
Remove-Item "HKCU:\Software\Microsoft\Edge\NativeMessagingHosts\com.iemode.host"   -Force -ErrorAction SilentlyContinue

# 删除 manifest 文件（路径为安装时生成的路径）
Remove-Item "<项目目录>\native\com.iemode.host.json" -Force -ErrorAction SilentlyContinue
```

删除 `<项目目录>` 文件夹即完成全部清理。
